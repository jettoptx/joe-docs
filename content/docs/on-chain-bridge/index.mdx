The **On-Chain Bridge** maps every path that connects OPTX to external chains. The core rule: all verification stays on Solana via the <span style={{color: "#eab308"}}>AARON protocol</span>.

> **Status:** OPTX flows are currently **Solana-only**. EVM and XRPL bridging are on the roadmap. Earlier versions of these docs described LayerZero OFT and Wormhole NTT pages — those have been retired pending re-implementation.

## Bridge Hub

<D2Diagram src="/diagrams/bridge-hub.svg" alt="On-Chain Bridge Hub — OPTX through AARON to Solana" caption="On-Chain Bridge Hub" />

| Node | Description |
|------|-------------|
| **OPTX** | $OPTX token — source of all bridge flows |
| **AARON** | AARON Protocol — Solana-native verification router |
| **SOL** | Solana — home chain, direct SPL / Token-2022 |

## Routing Logic

The <span style={{color: "#eab308"}}>AARON Router</span> on the Jetson edge node is the single choke point. It validates gaze biometrics, DePIN proofs, and AGT tensor attestations before any on-chain action. Once verified:

| Destination | Protocol | Asset Flow | Status |
|-------------|----------|-----------|--------|
| **Solana Native** | Direct SPL / Token-2022 | $OPTX, $JTX, $CSTB, Metaplex Core | Live |
| EVM / XRPL / Omnichain | — | Roadmap | Not currently routed |

## Key Delineation

<span style={{color: "#eab308"}}>**AARON Protocol**</span> is Solana-native. Gaze verification, biometric proofs, DePIN staking, and agent identity all happen on-chain via Helius RPC. This is the trust root.
