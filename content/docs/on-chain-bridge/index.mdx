The **On-Chain Bridge** maps every path that connects OPTX to external chains. The core rule: all verification stays on Solana via the <span style={{color: "#eab308"}}>AARON protocol</span>. Everything that leaves Solana passes through bridge infrastructure.

## Routing Logic

The <span style={{color: "#eab308"}}>AARON Router</span> on the Jetson edge node is the single choke point. It validates gaze biometrics, DePIN proofs, and AGT tensor attestations before any on-chain action. Once verified:

| Destination | Protocol | Asset Flow | Status |
|-------------|----------|-----------|--------|
| **Solana Native** | Direct SPL / Token-2022 | $OPTX, $JTX, $CSTB, Metaplex Core | Live |
| **EVM Chains** | LayerZero OFT | Wrapped $OPTX (ERC-20) | Devnet |
| **XRPL** | Wormhole NTT | USDC burn/mint, xSPECTAR NFTs | Live |
| **Omnichain** | LayerZero Messaging | Cross-chain DVN verification | Devnet |

## Key Delineation

**Native vs. Bridged**:

- <span style={{color: "#eab308"}}>**AARON Protocol**</span> is Solana-native. Gaze verification, biometric proofs, DePIN staking, and agent identity all happen on-chain via Helius RPC. This is the trust root.
- <span style={{color: "#60a5fa"}}>**Bridge Infrastructure**</span> handles everything that leaves Solana. LayerZero wraps $OPTX for EVM chains. Wormhole NTT burns/mints native USDC for XRPL flows. Neither bridge touches the verification layer.

The AARON router decides **what** gets bridged. The bridge protocols decide **how** it moves.
