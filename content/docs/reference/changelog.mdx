### Hermes v0.12.0 alignment

The OPTX deployment now tracks **Hermes Agent v0.12.0** (NousResearch tag `2026.4.30`) — *"The Curator release."* Released 2026-04-30 (1,096 commits, 550 PRs, 213 contributors since v0.11.0). Full feature breakdown in the new [Hermes v0.12.0 Features](/docs/astrojoe/hermes-features) page. Headline features that affect AstroJOE / JOE deployments on the [agent mesh](/docs/architecture/topology):

- **Autonomous Curator** — background process grades, prunes, and consolidates the skill library without operator intervention. Aligns with the existing AstroJOE self-maintenance protocol.
- **Self-improvement loop** substantially upgraded; review summaries surface in the TUI transcript and via `hermes curator status` (most-used + least-used skills).
- **Four new inference providers** added — broader fallback options on top of HEDGEHOG's Grok routing.
- **Microsoft Teams + Yuanbao** added as the 18th and 19th messaging platforms (joining the existing Matrix/X/etc. set).
- **Spotify + Google Meet** native integrations.
- **ComfyUI + TouchDesigner-MCP** bundled by default — these MCP servers are now first-class for AstroJOE on Jetson + CorsairOne without manual installation.
- **Humanizer skill** ported from earlier branches.
- **~57% TUI cold-start reduction** — perceptibly faster `hermes` launch on edge nodes.

Pinned version references in the docs site were unpinned in prose during this overhaul (link to [hermes-agent releases](https://github.com/NousResearch/hermes-agent/releases) instead). Where a version *is* required for a hard dependency claim, use **v0.12.0**.
